# M1 수집 파이프라인 실데이터 검증 노트

> 실행일: 2026-07-05 · 대상: 금융위 보도자료 RSS 최신 10건 (`POST /api/admin/ingest/run` 1회)
> 목적: 전체 흐름 동작 확인 + 형식/구조 리스크(특히 HWP) 파악. 코드 대규모 변경 아님.

## 1. 결과 요약

| 지표 | 값 |
|---|---|
| RSS 감지 | 10건 |
| 신규 적재(press_release) | 10건 (중복 0) |
| 본문 파싱 성공 / 실패 | **10 / 0** |
| 첨부 총계 | 29건 (다운로드 성공 29, 실패 0) |
| 첨부 타입 분포 | pdf 13, hwp 10, hwpx 5, **zip 1** |
| 본문 길이 범위 | 578 ~ 11,352자 |
| 담당부서(dept) 추출 | 10/10 성공 |

- 전 건이 PDF 우선 전략으로 본문 확보 → HWP 폴백은 **실제로 한 번도 호출되지 않음**.
- 별도로 다운로드된 hwp/hwpx **15건을 직접 파싱 스트레스 테스트 → 15/15 성공** (hwp: pyhwp, hwpx: zip/XML).
  현재 금융위 보도자료 형식에서는 HWP 파싱도 안정적.

## 2. 파싱 실패 사례

이번 배치에서는 **실패 0건**. 다만 아래는 잠재 리스크(오늘은 안 터졌지만 대비 필요).

## 3. 확인된 리스크 & 개선 제안

### R1. `zip` 첨부 미지원 (실제 관측됨)
- "연체초기 성실상환 채무자…" 건에 `.zip` 첨부 1건 존재. 파서는 pdf/hwp/hwpx만 처리 → zip은 조용히 무시.
- 이번엔 같은 건에 pdf가 있어 문제없었으나, **zip 안에 본문 hwp가 든 경우 본문 유실** 가능.
- 제안: `parsers`에 zip 핸들러 추가(압축 해제 후 내부 hwp/pdf 재귀 파싱) 또는 최소한 zip-only 시 `failed` 명시 + 경고.

### R2. pyhwp 포맷 프래그마 (잠재)
- 실 hwp 파싱 중 내부적으로 `undefined UnderlineStyle value: 15` 류의 미지원 enum을 만남(현재 비치명적, 로깅은 억제).
- 구형/변형 hwp에서 파싱 실패·부분누락 가능성. pyhwp는 유지보수가 활발하지 않음.
- 제안: 폴백 시 실패율 모니터링 지표화. 장기적으로 hwp5→(대안: libhwp, hwp5txt CLI, 또는 사내망에선 규칙기반) 이중화 검토.

### R3. 스캔/이미지 PDF → OCR 부재 (잠재, 설계 7.3)
- 이번 배치엔 없었으나, 텍스트 레이어 없는 PDF는 본문 0자 → `failed`.
- 제안: 본문 0자 + 첨부 존재 시 "스캔 의심" 플래그. OCR(tesseract)은 빈도 보고 결정.

### R4. 표(table) 내용 손실 (잠재)
- HWP 파싱 시 표는 `<표>` placeholder로 나오고, 텍스트 정리에서 제거됨 → 표 안 수치/항목 유실.
- 보도자료 핵심 수치가 표에 있는 경우 요약 품질 저하 가능(M2 영향).
- 제안: 표 셀 텍스트 추출 옵션 검토(PDF는 pdfplumber `extract_tables`, HWP는 별도).

### R5. dept/첨부 선택자 단일 의존 (구조 변경 취약)
- 담당부서: `div.board-view-wrap div.header div.info` + `<strong>담당부서</strong>` 단일 경로.
- 첨부: `div.file-list-wrap div.file-list > a`.
- 사이트 개편 시 조용히 빈 값 반환(현재 경고 로그만). 제안: 추출 0건 지속 시 알림(모니터링) + 선택자 회귀 테스트 픽스처 주기 갱신.

### R6. PDF 우선 전략이 HWP 회귀를 가림 (운영 관측성)
- 프로덕션에선 HWP 파서가 거의 안 돌아 회귀가 늦게 발견될 수 있음.
- 제안: 주기적으로 hwp/hwpx 단독 파싱 검증 잡(또는 CI 픽스처)으로 폴백 경로 상시 점검.

## 4. 다음 단계(M2) 참고
- 본문 품질은 양호(수천 자 확보). 요약·엔터티 추출 입력으로 충분.
- 단, 표 기반 수치(R4)·zip 본문(R1)은 M2 품질에 직접 영향 → 우선순위 높게 재검토.
- RSS 항목에 `<guid>`가 없어 링크 sha256 해시를 guid로 사용 중(중복키 안정적, 참고).
